## LAYER 1 — NORMATIVE (Constitutional Authority)

### Constitutional Intent

Why does this obligation matter educationally? What problem does it solve?

Reference the Immutable Law(s) and Constitutional Principle(s) operationalized.

---

### Constitutional Rationale

Why does this requirement exist constitutionally?

How does it operationalize Law and Principle? What downstream obligations does it enable?

---

### Constitutional Obligation

One sentence. Normative commitment only.

**Mandatory language:** shall, must, must not  
**Forbidden:** will, should, ought, may, normally, typically

No implementation parameters, percentages, effect sizes, time windows, research claims.

---

### Constitutional Invariants

Explicit guardrails. What must this requirement never do?

Formatted as bullet list. Each invariant is a "must never" commitment.

---

## LAYER 2 — INTERPRETIVE (Authoritative Guidance)

### Interpretation

How does this commitment apply in real situations? What does it mean operationally?

---

### Observable Behaviors

What would an observer see if this requirement were implemented?

Not outcomes. Behaviors. Not "students improve" but "system creates records."

---

### Audit Procedure

How do we verify this requirement was implemented?

Test behavior, not outcomes. Verify the system did what it committed to do.

---

### Compliance Criteria

Constitutional compliance checklist. Did the requirement exist and work?

Not product validation. Only: did the system implement what it committed?

---

### Implementation Boundary

What system owns this? What is constitutional scope?

Scope definition, not implementation advice.

---

### Failure Modes

How could this requirement be violated? What is the constitutional impact?

---

### Constitutional Logic Trace

Show the derivation chain: LAW-XXXX → PRIN-XXXX → REQ-XXXX → Implementation.

---

## LAYER 3 — INFORMATIVE (Non-Normative)

### Grounding Evidence

Research, cognitive science, educational evidence that informed this requirement.

Research informs constitutional design but does not constitute it.

---

### Implementation Notes

How might this requirement be implemented? What are practical considerations?

Not a specification. Trade-offs and engineering guidance.

---

### Depends On

What other requirements must be satisfied first?

---

### Enables

What downstream requirements depend on this?

---

### Potential Tensions

What educational values might conflict with this requirement?

How should they be resolved?

---

### Downstream Dependencies

What implementation systems depend on this requirement being met?

## TEMPLATE GOVERNANCE

### Mandatory Enforcement

Every requirement must:

✓ Include exactly one constitutional obligation  
✓ Use mandatory language only (shall/must/must not)  
✓ Exclude implementation parameters, percentages, effect sizes  
✓ Include Constitutional Invariants  
✓ Include Failure Modes  
✓ Include Constitutional Logic Trace  
✓ Move research claims to Grounding Evidence  
✓ Restrict Compliance Criteria to compliance  
✓ Restrict Audit Procedures to behavior  

### Enforcement Gates

Every requirement passes through:

1. **Template Conformance** — All sections present
2. **Atomicity & Language** — One obligation; mandatory language only
3. **Verification Feasibility** — Auditable, falsifiable, testable
4. **Glossary Consistency** — Terms match constitutional definitions
5. **Traceability & Overlap** — Traces to Law/Principle; unique owner
6. **Batch Approval** — All gates pass; internally consistent; ready for integration
