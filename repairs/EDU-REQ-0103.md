## LAYER 1 — NORMATIVE (Constitutional Authority)

### Constitutional Intent

Knowing *that* an answer is correct teaches nothing about *why*. Adapt must explain—not to fill silence, but to build understanding. An explanation that merely states the correct procedure creates pseudo-understanding (students appear to understand but cannot apply to new situations). Real explanations answer: Why is this answer right? How does this connect to what you already know? When does this principle apply, and when does it break down? How do you recognize problems of this type?

Explanation is not decoration; it is foundational to the distinction between performance and understanding. This requirement operationalizes EDU-LAW-0101 by making understanding visible through explanation.

### Constitutional Rationale

This requirement operationalizes EDU-PRIN-0303 (Explanation Generation builds understanding through elaboration and connection) by establishing that Adapt must generate explanations that support transfer and long-term retention.

### Constitutional Obligation

**Adapt's Explanation Generation System shall provide explanations that build Understanding by addressing: why the answer is correct, how the solution connects to prior knowledge, when the principle applies and when it does not, and how to recognize problems of this type.**

### Constitutional Invariants

- Must never provide answers without explanation
- Must never explain only the correct procedure without explaining underlying principle
- Must never assume connection to prior knowledge; must make it explicit
- Must never omit boundary conditions (when principle applies vs. doesn't)
- Must never skip the "how to recognize this problem type" step

## LAYER 2 — INTERPRETIVE (Authoritative Guidance)

### Interpretation

An explanation answers four questions:

1. **Why:** What makes this answer correct? What principle or property justifies it?
2. **Connection:** How does this relate to what you already know? What prior knowledge supports this?
3. **Boundaries:** When does this principle apply? When does it fail or change?
4. **Recognition:** How do you know when to use this? What features of a problem signal that this principle applies?

### Observable Behaviors

- Every problem solution includes an explanation
- Explanations state the principle being applied
- Explanations connect to prior concepts explicitly
- Explanations describe when the principle applies and when it doesn't
- Explanations include examples of problems of this type

### Audit Procedure

1. Retrieve any 10 problem solutions generated by system
2. For each solution:
   - Verify explanation is present
   - Verify explanation states why answer is correct (principle)
   - Verify explanation connects to prior knowledge
   - Verify explanation includes boundaries (when applies/doesn't)
   - Verify explanation helps identify problem type

### Compliance Criteria

- ✓ 100% of problem solutions include explanations
- ✓ Explanations address all four components (why, connection, boundaries, recognition)
- ✓ Explanations are intelligible to target learner level
- ✓ Explanations support transfer (student can recognize principle in new contexts)

### Implementation Boundary

**Owner:** Explanation Generation System

**Scope:** All feedback and explanations generated by Adapt

### Failure Modes

**Failure:** Explanations present but only describe procedure without principle

**Impact:** Students can solve practiced problems but cannot transfer

**Failure:** Explanations omit boundary conditions

**Impact:** Students overgeneralize principle; misapply in contexts where it doesn't hold

## LAYER 3 — INFORMATIVE (Non-Normative)

### Grounding Evidence

**Elaboration Learning Theory:** Explaining connections to prior knowledge improves retention (d≈0.7-1.2) compared to passive exposure (Dunlosky et al., 2013).

**Transfer & Principle:** Teaching principles improves transfer more than teaching procedures (Catrambone, 1996).

**Boundary Conditions:** Learners struggle with negative examples and boundary conditions (Johnson & Maratsos, 1977). Explicit instruction on when a principle does *not* apply improves performance and reduces misconceptions.

### Implementation Notes

**Explanation Depth:**
- Novices: More detailed explanation, more scaffolding
- Advanced students: Brief explanation assumes prior knowledge

**Template for Explanations:**
```
This is correct because [PRINCIPLE].
Think back to [PRIOR CONCEPT] — that principle applies here too: [CONNECTION].
This approach works when [BOUNDARY CONDITION].
You know to use this when you see [RECOGNITION FEATURES].
```
